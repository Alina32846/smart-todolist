# Умный список задач (Smart TodoList)

Клиент-серверное веб-приложение для ведения списка задач. Можно добавлять
задачи, отмечать их выполненными, удалять и фильтровать (все / активные /
завершённые). Фронтенд написан на React, бэкенд — на Express с REST API.

🔗 **Демо:** _ссылка появится после деплоя на Vercel_

## Возможности

- Добавление, удаление и отметка выполнения задач
- Фильтрация: все / активные / завершённые
- Счётчик активных задач
- Адаптивная вёрстка (корректно работает на телефоне и на десктопе)
- Обработка сетевых и серверных ошибок с возможностью повторить запрос

## Технологии

- **Frontend:** React 18, Vite, чистый CSS3 (адаптивная вёрстка)
- **Backend:** Node.js, Express, REST API
- **Тесты:** Jest + React Testing Library
- **Деплой:** Vercel (фронтенд как статика, бэкенд как serverless-функция)

## Запуск проекта локально

### 1. Установка зависимостей

```bash
npm install
```

### 2. Переменные окружения

Скопируйте файл `.env.example` в `.env` и задайте значение ключа
(значения `API_KEY` и `VITE_API_KEY` должны совпадать):

```bash
cp .env.example .env
```

```
API_KEY=dev-secret-key
VITE_API_KEY=dev-secret-key
```

### 3. Запуск в режиме разработки

```bash
npm run dev
```

Команда поднимает одновременно фронтенд (Vite, http://localhost:5173) и
бэкенд (Express, http://localhost:3001). Откройте в браузере
http://localhost:5173.

### Другие команды

```bash
npm test       # запустить тесты
npm run build  # собрать продакшен-версию фронтенда в папку dist
```

## Документация API

Базовый адрес: `/api`. Все запросы к задачам требуют заголовок
`x-api-key` со значением ключа из переменных окружения. Тело запросов и
ответы — в формате JSON.

### Модель задачи

```json
{
  "id": "строка (uuid)",
  "title": "текст задачи",
  "completed": false,
  "createdAt": "дата создания в формате ISO"
}
```

### Эндпоинты

| Метод | Путь | Описание | Тело запроса |
|-------|------|----------|--------------|
| GET | `/api/health` | Проверка работоспособности (без ключа) | — |
| GET | `/api/todos` | Получить список всех задач | — |
| POST | `/api/todos` | Создать задачу | `{ "title": "..." }` |
| PATCH | `/api/todos/:id` | Обновить задачу (частично) | `{ "completed": true }` или `{ "title": "..." }` |
| PUT | `/api/todos/:id` | Обновить задачу | то же, что у PATCH |
| DELETE | `/api/todos/:id` | Удалить задачу | — |

### Коды ответов

| Код | Когда возвращается |
|-----|--------------------|
| 200 | Успешный GET / PATCH / PUT |
| 201 | Задача создана (POST) |
| 204 | Задача удалена (DELETE) |
| 400 | Ошибка валидации тела запроса |
| 401 | Неверный или отсутствующий API-ключ |
| 404 | Задача с указанным id не найдена |
| 500 | Внутренняя ошибка сервера |

### Примеры запросов

Получить список задач:

```bash
curl -H "x-api-key: dev-secret-key" http://localhost:3001/api/todos
```

Создать задачу:

```bash
curl -X POST \
  -H "x-api-key: dev-secret-key" \
  -H "Content-Type: application/json" \
  -d '{"title":"Купить молоко"}' \
  http://localhost:3001/api/todos
```

Отметить задачу выполненной:

```bash
curl -X PATCH \
  -H "x-api-key: dev-secret-key" \
  -H "Content-Type: application/json" \
  -d '{"completed":true}' \
  http://localhost:3001/api/todos/<id>
```

Удалить задачу:

```bash
curl -X DELETE \
  -H "x-api-key: dev-secret-key" \
  http://localhost:3001/api/todos/<id>
```

## Деплой

Проект разворачивается на Vercel: фронтенд собирается командой `vite build`
и отдаётся как статика, а папка `api/` автоматически становится
serverless-функцией. Маршрутизация `/api/*` настроена в `vercel.json`.
Значения `API_KEY` и `VITE_API_KEY` задаются в настройках проекта на Vercel
(в репозиторий ключи не попадают). Деплой запускается автоматически при
каждом push в ветку `main`.
